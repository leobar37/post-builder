# Database Schema

## Overview

SQLite database for metadata storage. Videos are stored in filesystem, paths referenced in DB.

## Tables

### projects

Projects contain multiple videos and define the context folder for OpenCode.

```sql
CREATE TABLE projects (
  id TEXT PRIMARY KEY,                    -- UUID v4
  name TEXT NOT NULL,                     -- Project name (e.g., "Promo Mayo 2024")
  description TEXT,                       -- Project description
  context_id TEXT NOT NULL,               -- Folder name in projects/ (e.g., "promo-mayo-2024")
  
  -- Project configuration (JSON for flexibility)
  config TEXT,                            -- JSON: { theme, defaultDuration, etc. }
  
  -- Status
  status TEXT DEFAULT 'active',           -- active | archived | deleted
  
  -- Timestamps
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**Indexes:**
```sql
CREATE INDEX idx_projects_status ON projects(status);
CREATE INDEX idx_projects_created ON projects(created_at DESC);
```

### videos

Main table for video projects. Each video belongs to a project.

```sql
CREATE TABLE videos (
  id TEXT PRIMARY KEY,                    -- UUID v4
  project_id TEXT NOT NULL,               -- FK to projects
  status TEXT NOT NULL,                   -- Status enum
  prompt TEXT NOT NULL,                   -- User prompt
  
  -- Note: context is now inherited from project
  -- context_path is resolved from project at runtime
  
  -- OpenCode session tracking
  opencode_session_id TEXT,               -- ACP session ID
  opencode_status TEXT,                   -- Session status
  
  -- Video idea (generated by OpenCode)
  idea_title TEXT,
  idea_description TEXT,
  idea_json TEXT,                         -- JSON array of scenes
  
  -- Contexto específico del video (JSON)
  context TEXT,                           -- VideoContext: tema, objetivo, estilo
  
  -- Final output
  output_path TEXT,                       -- Path to final MP4
  output_url TEXT,                        -- Public URL
  
  -- Progress tracking
  total_scenes INTEGER DEFAULT 0,
  completed_scenes INTEGER DEFAULT 0,
  progress INTEGER DEFAULT 0,             -- 0-100
  
  -- Timestamps
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  completed_at DATETIME
);
```

**Indexes:**
```sql
CREATE INDEX idx_videos_project ON videos(project_id);
CREATE INDEX idx_videos_status ON videos(status);
CREATE INDEX idx_videos_created ON videos(created_at DESC);
CREATE INDEX idx_videos_session ON videos(opencode_session_id);
```

**Foreign Key:**
```sql
FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
```

### scenes

Individual scenes/clips within a video.

```sql
CREATE TABLE scenes (
  id TEXT PRIMARY KEY,                    -- UUID v4
  video_id TEXT NOT NULL,
  
  -- Scene metadata
  sequence INTEGER NOT NULL,              -- Order in video (0, 1, 2...)
  description TEXT,                       -- Scene description from OpenCode
  duration INTEGER DEFAULT 6,             -- Seconds (6, 8, 10)
  
  -- MiniMax generation
  minimax_task_id TEXT,                   -- MiniMax API task ID
  minimax_status TEXT,                    -- queued|processing|success|fail
  minimax_prompt TEXT,                    -- Text-to-video prompt
  
  -- Contexto específico de la escena (JSON)
  context TEXT,                           -- SceneContext: acción, elementos, texto
  
  -- Output
  clip_path TEXT,                         -- Local path to MP4
  clip_url TEXT,                          -- Public URL
  file_id TEXT,                           -- MiniMax file ID
  
  -- Error tracking
  error_message TEXT,
  retry_count INTEGER DEFAULT 0,
  
  -- Timestamps
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  generated_at DATETIME,
  
  FOREIGN KEY (video_id) REFERENCES videos(id) ON DELETE CASCADE
);

**Indexes:**
```sql
CREATE INDEX idx_videos_project ON videos(project_id);
CREATE INDEX idx_scenes_video ON scenes(video_id);
CREATE INDEX idx_scenes_sequence ON scenes(video_id, sequence);
CREATE INDEX idx_scenes_minimax ON scenes(minimax_task_id);
CREATE INDEX idx_scenes_status ON scenes(minimax_status);
```

### events

Event log for debugging and replay.

```sql
CREATE TABLE events (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  video_id TEXT,
  session_id TEXT,
  
  -- Event data
  type TEXT NOT NULL,                     -- Event type enum
  source TEXT NOT NULL,                   -- opencode|minimax|system
  data TEXT,                              -- JSON payload
  
  -- Metadata
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (video_id) REFERENCES videos(id) ON DELETE CASCADE
);
```

**Indexes:**
```sql
CREATE INDEX idx_events_video ON events(video_id, created_at DESC);
CREATE INDEX idx_events_session ON events(session_id, created_at DESC);
```

## Status Enums

### Video Status

```typescript
type VideoStatus =
  | 'draft'              // Initial state
  | 'generating_idea'    // OpenCode generating idea
  | 'idea_ready'         // Idea ready, waiting for approval
  | 'generating_clips'   // MiniMax generating clips
  | 'clips_ready'        // All clips ready
  | 'composing'          // Remotion composing
  | 'completed'          // Final video ready
  | 'failed';            // Error occurred
```

### Scene Status (MiniMax)

```typescript
type SceneStatus =
  | 'pending'      // Waiting to start
  | 'queued'       // In MiniMax queue
  | 'processing'   // Generating
  | 'success'      // Completed
  | 'fail'         // Failed
  | 'retrying';    // Automatic retry
```

## Sample Queries

### Get all projects with video count

```sql
SELECT 
  p.*,
  COUNT(v.id) as video_count,
  COUNT(CASE WHEN v.status = 'completed' THEN 1 END) as completed_videos
FROM projects p
LEFT JOIN videos v ON v.project_id = p.id
WHERE p.status = 'active'
GROUP BY p.id
ORDER BY p.created_at DESC;
```

### Get project with videos

```sql
SELECT p.*,
       json_group_array(
         json_object(
           'id', v.id,
           'status', v.status,
           'prompt', v.prompt,
           'progress', v.progress,
           'created_at', v.created_at
         ) ORDER BY v.created_at DESC
       ) as videos_json
FROM projects p
LEFT JOIN videos v ON v.project_id = p.id
WHERE p.id = ?
GROUP BY p.id;
```

### Get video with scenes

```sql
SELECT v.*, 
       json_group_array(
         json_object(
           'id', s.id,
           'sequence', s.sequence,
           'status', s.minimax_status,
           'clip_url', s.clip_url
         ) ORDER BY s.sequence
       ) as scenes_json
FROM videos v
LEFT JOIN scenes s ON s.video_id = v.id
WHERE v.id = ?
GROUP BY v.id;
```

### Update progress

```sql
-- Update video progress based on completed scenes
UPDATE videos
SET progress = (
  SELECT ROUND(
    (COUNT(CASE WHEN minimax_status = 'success' THEN 1 END) * 100.0) / COUNT(*)
  )
  FROM scenes WHERE video_id = videos.id
),
completed_scenes = (
  SELECT COUNT(*) FROM scenes 
  WHERE video_id = videos.id AND minimax_status = 'success'
),
updated_at = CURRENT_TIMESTAMP
WHERE id = ?;
```

### Recent events for video

```sql
SELECT * FROM events
WHERE video_id = ?
ORDER BY created_at DESC
LIMIT 100;
```

### Query videos by context theme

```sql
-- Buscar videos por tema en contexto JSON
SELECT id, name, prompt
FROM videos
WHERE json_extract(context, '$.theme') LIKE '%verano%'
ORDER BY created_at DESC;
```

### Query scenes by emotion

```sql
-- Buscar escenas por emoción en contexto JSON
SELECT s.id, s.sequence, s.description, v.id as video_id
FROM scenes s
JOIN videos v ON v.id = s.video_id
WHERE json_extract(s.context, '$.emotion') = 'excitement'
ORDER BY s.created_at DESC;
```
