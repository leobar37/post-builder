1 Opción 1: ACP (Agent Client Protocol)

Comunicación bidireccional persistente via stdin/stdout usando JSON-RPC. El proceso OpenCode se mantiene vivo durante toda la sesión.

Tu API - JSON-RPC Client TypeScript 
import { spawn } from 'child_process';

class OpenCodeACPClient {
  private process: ChildProcess;
  private requestId = 0;
  private pendingRequests = new Map();

  async connect() {
    // Inicia OpenCode en modo ACP
    this.process = spawn('opencode', ['acp', '--cwd', './project']);

    // Lee respuestas JSON-RPC de stdout
    this.process.stdout?.on('data', (data) => {
      const messages = data.toString().split('\n').filter(Boolean);
      for (const msg of messages) {
        const response = JSON.parse(msg);
        this.handleResponse(response);
      }
    });
  }

  async sendPrompt(prompt: string) {
    const id = ++this.requestId;
    const request = {
      jsonrpc: '2.0',
      id,
      method: 'session/prompt',
      params: { prompt }
    };

    this.process.stdin?.write(JSON.stringify(request) + '\n');

    return new Promise((resolve) => {
      this.pendingRequests.set(id, resolve);
    });
  }
}

✓ Ventajas

  • • Sesión persistente con contexto
  • • Streaming de respuestas en tiempo real
  • • OpenCode puede leer/escribir archivos directamente
  • • Comunicación bidireccional

✗ Desventajas

  • • Complejo de implementar (necesitas cliente JSON-RPC)
  • • Proceso debe mantenerse vivo
  • • Manejo de estado complicado
  • • Reconexiones difíciles

2 Opción 2: CLI Run Mode

Ejecuta OpenCode con un prompt específico, espera a que termine y captura el resultado. Cada ejecución es independiente.

Comando básico Bash 
opencode run "Genera una idea de video para promoción de mayo" \
  --file ./content/promos \
  --format json \
  --title "video-gen-001"
Con servidor attachado (evita cold start) Bash 
# Terminal 1: Start server
opencode serve --port 4096

# Terminal 2: Run commands
opencode run "Genera idea de video" --attach http://localhost:4096
Tu API - Ejecución TypeScript 
import { execa } from 'execa';

export async function generateVideoIdea(
  prompt: string,
  contextPath: string
): Promise<VideoIdea> {
  const result = await execa('opencode', [
    'run',
    '--format', 'json',
    '--file', contextPath,
    '--title', `video-gen-${Date.now()}`,
    prompt
  ], {
    cwd: process.cwd(),
    timeout: 120000 // 2 minutos
  });

  return JSON.parse(result.stdout);
}

✓ Ventajas

  • • Simple de implementar
  • • Output JSON parseable
  • • No requiere mantener proceso vivo
  • • Cada llamada es aislada

✗ Desventajas

  • • No hay streaming en tiempo real
  • • Cada llamada es stateless
  • • Cold start lento (usa --attach)
  • • Timeout puede ser problemático

3 Opción 3: Skill + HTTP (Recomendado)

Creas una skill de OpenCode que se comunica con tu API via HTTP. La skill actúa como puente entre OpenCode y tu sistema.

Estructura de la Skill File Tree 
.opencode/
└── skills/
    └── video-pipeline/
        ├── skill.yaml
        ├── agent.md
        └── scripts/
            ├── generate.sh
            └── upload.sh
skill.yaml YAML 
name: video-pipeline
description: |
  Skill para integración con Video
  Pipeline System
tools:
  - http_request
endpoints:
  api: http://localhost:3458/api

Flujo Completo Recomendado

1

Usuario escribe prompt en Web UI

2

Tu API crea registro en SQLite (estado: draft)

3

Tu API ejecuta: opencode run --agent video-pipeline "..."

4

Skill lee contexto de paths en config.yaml

5

Skill hace POST a tu API con la idea generada

6

Tu API actualiza SQLite y genera scenes

7

Tu API inicia generación MiniMax para cada scene

8

Progreso se muestra en Web UI via polling

✓ Ventajas

  • • Skill reutilizable en cualquier proyecto OpenCode
  • • Comunicación HTTP estándar
  • • Fácil de debuggear (logs HTTP)
  • • Abstracción limpia
  • • Desacoplado del core de tu API

⚠ Consideraciones

  • • Necesitas crear y mantener la skill
  • • Latencia adicional por HTTP calls
  • • Requiere entender el skill system de OpenCode
  • • Versión de skill debe sincronizarse

Endpoints API Necesarios

GET /api/config Obtener config

Retorna paths y modelos configurados en config.yaml

POST /api/videos Crear video

Crea nuevo video (prompt + context_key)

GET /api/videos/:id Estado

Retorna estado del video y sus clips

POST /api/videos/:id/idea Recibir idea

Skill POSTea aquí la idea generada

POST /api/videos/:id/regenerate Re-generar

Re-generar clip específico

GET /api/videos/:id/preview Preview

URL para Remotion Player

api/services/opencode-service.ts TypeScript 
import { execa } from 'execa';
import { getConfig } from './config';

export class OpenCodeService {
  async generateVideoIdea(
    videoId: string,
    prompt: string,
    contextKey: string
  ): Promise<void> {
    const config = getConfig();
    const contextPath = config.context.paths[contextKey];

    // Ejecuta OpenCode con la skill video-pipeline
    const result = await execa('opencode', [
      'run',
      '--agent', 'video-pipeline',
      '--format', 'json',
      '--title', `video-${videoId}`,
      `Genera idea de video para: ${prompt}. ` +
      `Contexto en: ${contextPath}. ` +
      `POST resultado a /api/videos/${videoId}/idea`
    ], {
      cwd: process.cwd(),
      env: {
        ...process.env,
        VIDEO_PIPELINE_API_URL: 'http://localhost:3458',
        VIDEO_ID: videoId
      },
      timeout: 180000 // 3 minutos
    });

    console.log('OpenCode result:', result.stdout);
  }
}
.opencode/skills/video-pipeline/scripts/generate.sh Bash 
#!/bin/bash
# Script llamado por la skill para comunicar resultado a la API

set -e

API_URL="${VIDEO_PIPELINE_API_URL:-http://localhost:3458}"
VIDEO_ID="${VIDEO_ID}"
IDEA_JSON="${1}"

# Enviar idea generada a la API
curl -X POST "${API_URL}/api/videos/${VIDEO_ID}/idea" \
  -H "Content-Type: application/json" \
  -d "${IDEA_JSON}" \
  --silent \
  --show-error

echo "Idea enviada a API exitosamente"
