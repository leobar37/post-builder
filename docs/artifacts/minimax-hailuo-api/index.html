¿Qué es Hailuo AI?

Es el modelo de generación de video de MiniMax. Tiene 4 modos:

  • Text-to-Video: Escribes un prompt, genera video
  • Image-to-Video: Subes una imagen + prompt, anima la imagen
  • First-Last-Frame: Subes imagen inicial + imagen final, genera transición
  • Subject Reference: Subes una foto de cara + prompt, mantiene la identidad facial

✍️ Parte 1: Cómo Escribir Prompts que Funcionan

🎯 La Fórmula Básica (Text-to-Video)

[SUJETO] + [ACCIÓN/MOVIMIENTO] + [ENTORNO] + [CÁMARA] + [ESTILO/ATMÓSFERA]
✓ BUENO
"A professional ballerina performing a grand jeté leap in a sunlit Parisian studio with large windows. [slow-motion] The camera follows her movement from a low angle, capturing dust particles in the golden hour light. Elegant, cinematic, film grain texture."
✗ MALO
"a girl dancing"
💡 CLAVE
Sé específico. "Dancing" es vago. "Performing a grand jeté leap" es preciso. La IA necesita detalles concretos para entender qué movimiento quieres.

🎥 Comandos de Cámara (tags especiales)

Hailuo entiende estos tags entre corchetes para controlar el movimiento de cámara:

Tag Qué hace Ejemplo de uso
[pan] Paneo horizontal (izquierda/derecha) "[pan left] across the mountain landscape"
[zoom] Zoom in/out "[zoom in] slowly on her face"
[static] Cámara fija, sin movimiento "[static] shot of the burning candle"
[tilt] Inclinación vertical "[tilt up] from shoes to face"
[tracking] Cámara sigue al sujeto "[tracking] the running athlete"
[slow-motion] Efecto cámara lenta "[slow-motion] water droplet falling"
✓ Ejemplo completo con cámara
"A samurai warrior drawing his katana in a bamboo forest at dawn. [tracking] his movement as he spins. [zoom in] on his determined eyes. Morning mist, rays of sunlight filtering through bamboo, cinematic composition, 35mm film look."

🖼️ Image-to-Video: Cómo animar una imagen

Cuando usas Image-to-Video, el prompt describe cómo debe moverse/evolucionar la escena desde tu imagen estática.

✗ ERROR COMÚN
Subes foto de una mujer sentada
Prompt: "a woman sitting"

Resultado: No pasa nada, la imagen está estática
✓ CORRECTO
Subes foto de una mujer sentada
Prompt: "The woman slowly lifts her head and gazes out the window, her hair gently moving in the breeze. [slow-motion] [pan right] to reveal the rainy cityscape outside. Melancholic atmosphere, soft natural lighting."

Resultado: La mujer se mueve, la cámara panea
Regla de oro: El prompt debe describir movimiento y cambio, no solo repetir lo que hay en la imagen.

🎭 Subject Reference: Mantener la misma cara

Para que el personaje mantenga su identidad facial a través de diferentes escenas:

Requisitos de la foto de referencia:
  • Una sola persona (no grupos)
  • Rostro claro, sin gafas de sol ni máscaras
  • Mínimo 120x120px, máximo 20MB
  • Ángulo: de frente o ligeramente de lado (evitar perfil puro)
  • Iluminación natural, sin sombras duras en la cara
✓ EJEMPLO SUBJECT REFERENCE
Foto: Selfie clara de tu cara
Prompt: "In a cozy coffee shop, reading a book by the window. [static] shot, warm afternoon light, steam rising from the coffee cup. The subject occasionally looks up and smiles gently, then returns to reading. Documentary style, shallow depth of field."

🔌 Parte 2: Cómo Usar la API (Paso a Paso)

📋 Resumen del flujo

La API es asíncrona (no devuelve el video inmediatamente):

  1. Envías tu request → Recibes un task_id
  2. Preguntas cada 10 segundos: "¿Ya está listo?"
  3. Recibes file_id cuando termina
  4. Descargas el video usando el file_id
⚠️ Diferencia importante:
• Video normal → devuelve file_id → necesitas llamar a /files/retrieve
• Templates → devuelve directamente video_url (más fácil)

🐍 Código Python completo (Text-to-Video)

# 1. CONFIGURACIÓN
import os, time, requests

api_key = os.environ["MINIMAX_API_KEY"]  # Nunca hardcodees la key
headers = {"Authorization": f"Bearer {api_key}"}

# 2. CREAR LA TAREA
def crear_video():
    url = "https://api.minimax.io/v1/video_generation"
    
    payload = {
        "model": "MiniMax-Hailuo-2.3",  # Modelo más reciente
        "prompt": "A professional chef plating a gourmet dish in a Michelin-star kitchen. [slow-motion] [zoom in] on the delicate sauce drizzle. Steam rising, warm lighting, shallow depth of field, cinematic.",
        "duration": 6,  # 5, 6, o 10 segundos
        "resolution": "1080P"  # 480P, 720P, 1080P
    }
    
    response = requests.post(url, headers=headers, json=payload)
    data = response.json()
    
    return data["task_id"]  # Guarda este ID

# 3. ESPERAR A QUE TERMINE (POLLING)
def esperar_video(task_id):
    url = "https://api.minimax.io/v1/query/video_generation"
    
    while True:
        response = requests.get(url, headers=headers, 
                               params={"task_id": task_id})
        data = response.json()
        
        status = data["status"]
        print(f"Estado: {status}")
        
        if status == "Success":
            return data["file_id"]  # Éxito!
        elif status == "Fail":
            raise Exception("Falló la generación")
        
        time.sleep(10)  # Espera 10 segundos entre consultas

# 4. DESCARGAR EL VIDEO
def descargar_video(file_id):
    # Obtener URL de descarga
    url = "https://api.minimax.io/v1/files/retrieve"
    response = requests.get(url, headers=headers, 
                           params={"file_id": file_id})
    
    download_url = response.json()["file"]["download_url"]
    
    # Descargar el archivo
    video = requests.get(download_url)
    
    with open("mi_video.mp4", "wb") as f:
        f.write(video.content)
    
    print("✓ Video descargado")

# EJECUTAR TODO
task_id = crear_video()
print(f"Task creado: {task_id}")

file_id = esperar_video(task_id)
print(f"Completado! File ID: {file_id}")

descargar_video(file_id)

🎨 Código para Templates (más fácil)

Los templates son escenas predefinidas. Solo necesitas subir una imagen (y a veces texto).

# Ejemplo: Template "Run for Life"
# Tu mascota huyendo de un león en la sabana

def usar_template():
    url = "https://api.minimax.io/v1/video_template_generation"
    
    payload = {
        # ID del template "Run for Life"
        "template_id": "393769180141805569",
        
        # Imagen de tu mascota
        "media_inputs": [{
            "value": "https://tuservidor.com/tu-mascota.jpg"
        }],
        
        # Qué bestia la persigue
        "text_inputs": [{
            "value": "Lion"  # O Tiger, Bear, Wolf, etc.
        }]
    }
    
    response = requests.post(url, headers=headers, json=payload)
    return response.json()["task_id"]

# Consultar estado (nota: devuelve video_url directamente!)
def consultar_template(task_id):
    url = "https://api.minimax.io/v1/query/video_template_generation"
    
    while True:
        response = requests.get(url, headers=headers,
                               params={"task_id": task_id})
        data = response.json()
        
        if data["status"] == "Success":
            # Templates devuelven URL directa!
            return data["video_url"]  # No necesitas file_id
        
        time.sleep(10)

# Descargar directamente desde la URL
video_url = consultar_template(task_id)
video = requests.get(video_url)
with open("output.mp4", "wb") as f:
    f.write(video.content)
✓ Ventaja de Templates: No necesitas llamar a /files/retrieve. La consulta de estado te devuelve directamente la URL del video.

⚡ Resumen Rápido

Checklist antes de enviar un prompt:

  • ¿Describí el sujeto específicamente? (no solo "a person", sino "a elderly woman in a red coat")
  • ¿Incluí movimiento/acción concreto? (no solo "dancing", sino "spinning with arms extended")
  • ¿Agregué tags de cámara? ([zoom], [pan], [slow-motion])
  • ¿Describí el entorno? (lugar, iluminación, atmósfera)
  • ¿Mencioné el estilo? (cinematic, documentary, anime, etc.)
