Decisiones Arquitectónicas Clave

Context Convention: projects/{contextId}/

OpenCode siempre ejecuta desde workspace/ y accede al contexto mediante referencias a carpetas estandarizadas. Elimina problemas de permisos y paths arbitrarios.

ACP sobre CLI

Agent Client Protocol permite sesiones persistentes, streaming bidireccional y múltiples tool calls. Más complejo pero necesario para el pipeline interactivo.

Contexto Jerárquico de 3 Niveles

Project (archivos) → Video (JSON) → Scene (JSON). OpenCode lee archivos directamente; video/scene se pasan via ACP. Permite granularidad y reutilización.

Session Manager Robusto

Metadata completa (PID, status, lastMessage), health checks periódicos, tracking de mensajes. Permite recuperación y monitoreo del estado de sesiones ACP.

Selección de Modelos por Tarea

Idea→Opus (creatividad), Code→Sonnet (balance), Editing→Haiku (velocidad). Optimiza costo/rendimiento según el tipo de operación.

SSE sobre WebSocket

Server-Sent Events para streaming unidireccional (servidor→cliente). Más simple, compatible con HTTP/2, reconexión automática. Suficiente para el flujo de eventos.

  • visualStyle (estilo)
  • targetAudience
JSON enviado via ACP context

  • action (acción)
  • visualElements

interface SessionMetadata {
  id: string;                    // UUID de sesión
  contextId: string;             // projects/{contextId}/
  pid?: number;                  // Process ID
  status: 'running' | 'completed' | 'error';
  lastMessage?: string;          // Último mensaje
  messageCount: number;          // Contador
  startedAt: Date;               // Inicio
  lastActivityAt: Date;          // Última actividad
}
