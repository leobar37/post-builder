Integración con React 

function CanvasComponent() {
  const canvasRef = useRef<HTMLCanvasElement>(null);

  useEffect(() => {
    const canvas = canvasRef.current;
    if (!canvas) return;

    const ctx = canvas.getContext('2d');
    if (!ctx) return;

    // Lógica de dibujo
    ctx.fillStyle = '#3498db';
    ctx.fillRect(10, 10, 100, 100);
  }, []);

  return <canvas ref={canvasRef} />;
}
Patrón clave: Usar refs para datos de nodos (evita re-renders). Solo estado React para UI controls.

Patrones de Node Editors

Arquitectura para editores visuales basados en nodos

Estructura de Datos Recomendada

Estado normalizado (flat) para performance:

interface GraphState {
  // Entidades indexadas por ID
  nodes: Record<string, Node>;
  edges: Record<string, Edge>;

  // Arrays de IDs para orden
  nodeIds: string[];
  edgeIds: string[];

  // Estado UI separado
  ui: {
    selectedNodeIds: string[];
    viewport: { x, y, zoom };
  };
}
Nodo típico:
{ id, type, position: {x, y}, data, width, height, parentId }
Edge (conexión):
{ id, source, target, sourceHandle, targetHandle, type }
Handles / Ports:

Puntos de conexión en los nodos. Pueden ser parte del nodo o entidades independientes.

API Principal

import { ReactFlow,
  Background, Controls, MiniMap
} from '@xyflow/react';

function Flow() {
  const [nodes, setNodes] = useState(initialNodes);
  const [edges, setEdges] = useState(initialEdges);

  const onNodesChange = useCallback(
    (changes) => setNodes((nds) => applyNodeChanges(changes, nds)),
    []
  );

  const onConnect = useCallback(
    (params) => setEdges((eds) => addEdge(params, eds)),
    []
  );

  return (
    <ReactFlow
      nodes={nodes}
      edges={edges}
      onNodesChange={onNodesChange}
      onConnect={onConnect}
      fitView
    >
      <Background />
      <Controls />
      <MiniMap />
    </ReactFlow>
  );
}
Componentes Incluidos
  • • ReactFlow - Componente raíz con viewport
  • • Background - Grid/retícula configurable
  • • Controls - Botones zoom y fit view
  • • MiniMap - Vista miniatura del diagrama
  • • Panel - Contenedor flotante personalizado
Custom Node Types

Cualquier componente React puede ser un nodo. Recibe props inyectadas: id, data, selected, dragging.

Limitaciones Importantes

Escala

NO diseñado para 1000+ nodos. Requiere virtualización manual para datasets grandes.

Layout Automático

NO incluido nativamente. Requiere integrar Dagre (~40KB) o ELK (~1.4MB).

Edge Routing

No tiene routing automático para evitar nodos. Usar ELK para esto.

Usar React Flow cuando:

  • Necesitas un MVP funcional en días, no semanas
  • Los nodos necesitan UI compleja (formularios, botones)
  • Tu equipo tiene expertise React pero no Canvas
  • Esperas menos de 500 nodos en el editor
  • Estás validando un concepto (MVP)

Usar Canvas Nativo cuando:

  • Necesitas 1000+ nodos con buena performance
  • Animaciones complejas requeridas (60fps)
  • Bundle size es extremadamente restrictivo
  • Experiencia gaming/interactiva
  • Exportación vectorial perfecta (diagramas técnicos)

Conclusión

Para empezar a aprender, React Flow es la elección pragmática. Te permite entender los patrones de node editors sin lidiar con la complejidad de Canvas.

¿React Flow? Si quieres un editor funcional rápidamente con UI rica en los nodos.
¿Canvas Nativo? Si necesitas control total, 1000+ nodos, o animaciones complejas a 60fps.
