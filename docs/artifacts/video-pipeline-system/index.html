Pipeline de Generación (Web Flow)

1

Usuario en Web

UI

El usuario escribe un prompt y selecciona el contexto predefinido desde un dropdown.

Prompt: "Video promocional de membresías"

Contexto: promos (resuelto desde config)

2

API + Agente

Backend

La API lee el config, carga el contexto y el agente genera la idea y secuencia.

Lee config.yaml

paths.context.promos

Guarda en SQLite

video + scenes

3

MiniMax Generation

Async

Se envían los prompts a MiniMax API. Cada clip se genera en paralelo.

POST https://api.minimax.io/v1/video_generation
prompt: "Cinematic gym shot..."
model: "MiniMax-Hailuo-02"
duration: 8
4

Download + Storage

Files

Cuando MiniMax completa, se descarga el video al filesystem y se actualiza SQLite.

# Download
GET /v1/files/{file_id} → ./clips/scene-01.mp4

# Update DB
UPDATE scenes SET status='completed', path='...'
5

Preview + Export

Final

Remotion Player carga los clips generados para preview. Export genera MP4 final.

config.yaml

# Rutas de contexto predefinidas
context:
  paths:
    promos: "./content/promos"
    brand: "./content/brand"
    tutorials: "./content/tutorials"
    social: "./content/social"

models:
  default: "minimax-hailuo-02"
  available:
    - "minimax-hailuo-02"
    - "minimax-hailuo-2.3"

generation:
  defaults:
    duration: 8
    resolution: "1080p"
    aspectRatio: "9:16"
  limits:
    maxClips: 6
    maxPromptLength: 500

output:
  basePath: "./outputs"
  formats: ["mp4", "webm"]

SQLite Schema

-- Videos (metadatos)
CREATE TABLE videos (
  id TEXT PRIMARY KEY,
  prompt TEXT NOT NULL,
  context_key TEXT,
  status TEXT, -- draft|generating|completed
  total_clips INTEGER,
  created_at DATETIME,
  completed_at DATETIME
);

-- Scenes/Clips
CREATE TABLE scenes (
  id TEXT PRIMARY KEY,
  video_id TEXT,
  sequence INTEGER,
  name TEXT,
  prompt TEXT,
  status TEXT, -- pending|queued|generating|completed|failed
  task_id TEXT, -- MiniMax task_id
  file_id TEXT, -- MiniMax file_id
  local_path TEXT,
  error_message TEXT,
  created_at DATETIME
);

-- Generation logs
CREATE TABLE generations (
  id TEXT PRIMARY KEY,
  scene_id TEXT,
  provider TEXT, -- 'minimax'
  task_id TEXT,
  status TEXT,
  started_at DATETIME,
  completed_at DATETIME
);

Prompt Engineering para MiniMax

✓ Buenas prácticas

  • • Incluir movimiento de cámara: [dolly in], [handheld], [static]
  • • Especificar estilo: "cinematic shot", "8k quality"
  • • Describir iluminación: "golden hour", "soft lighting"
  • • Mantener prompts concisos (300-500 chars)

✗ Evitar

  • • Texto o logos (MiniMax no genera texto legible)
  • • Rostros específicos o personas identificables
  • • Prompts muy largos (>500 caracteres)
  • • Contenido con derechos de autor
